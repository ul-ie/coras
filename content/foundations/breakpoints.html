Breakpoints are foundational to responsive design. They are pre-defined values that determine how your layout behaves and adapts to different device and viewport sizes.

Mobile-First Design

Whether you're a developer, designer or content editor, when you are creating anything for a digital audience, you should first consider how it will appear and function on a mobile device, and only afterwards consider how it will adapt to larger devices or viewports.

Available Breakpoints

Our design system works with five breakpoints, defined by a series of minimum widths. What this means in practice is that breakpoints move upwards, from mobile (where no breakpoint is needed) to tablet, laptop, desktop monitor, widescreen etc.

Breakpoint prefix Minimum width CSS
sm 640px @media (min-width: 640px) { ... }
md 768px @media (min-width: 768px) { ... }
lg 1024px @media (min-width: 1024px) { ... }
xl 1280px @media (min-width: 1280px) { ... }
2xl 1536px @media (min-width: 1536px) { ... }

Using Breakpoints

For designers, ensure the frame widths defined in Figma correspond to the above breakpoint values. Ideally, use these values as variables in Figma. Plugins also exist which facilitate automatic responsive layouts.

For developers, breakpoints can be used as modifiers, prefixing normal style classes. This will ensure the styles in question only become active at the prefixed breakpoint(s).
