This section covers how to position and display elements, and how to best layout items in grids and container elements.

Display

Use the below utility classes to set the display property of elements.

Note: To prevent conflicts with Drupal, the block utility class has been removed from Tailwind and replaced with d-block.

Class Description
d-block Sets display to block
inline-block Sets display to inline-block
inline Sets display to inline
flex Sets display to flex
inline-flex Sets display to inline-flex
grid Sets display to grid
inline-grid Sets display to inline-grid

Visibility

When hiding an element, care should be taken to ensure that the correct class is chosen.

Class Description
hidden Sets display to none, removing it from view and screen readers
sr-only Removes an element visually without hiding it from screen readers

If an element still needs to be readable by assistive technology like screen readers, use sr-only. This does the same job as the visually-hidden class provided by Drupal, but the former is preferable as it is in our own codebase, is more concise and makes the meaning clearer.

The aria-hidden attribute should also be used where appropriate to hide something that only makes sense to a sighted individual. For example, a button icon should set aria-hidden="true".

Containers

Using the container class will centre the elements it contains (in a similar way to using mx-auto) and ensure the width never exceeds that of the currently active breakpoint (see breakpoints).

Max-Width

Whether you use a container or not, you might want to set the maximum width of an element to a specific breakpoint, or, for example, if the content contains prose, set it according to a maximum character count to aid legibility and accessibility. The following utility classes can prove useful for these situations.

Class CSS
max-w-prose max-width: 65ch;
max-w-screen-sm max-width: 640px;
max-w-screen-md max-width: 768px;
max-w-screen-lg max-width: 1024px;
max-w-screen-xl max-width: 1280px;
max-w-screen-2xl max-width: 1536px;

Flexbox vs Grid

Our layout system uses both Flexbox and CSS Grid, depending on the use case. The same layouts can generally be achieved using either, so the below are just general guidelines.

Choose Flexbox if:

  • You have just a few items to arrange
  • Centering items when they wrap onto a new row is important

Choose Grid if:

  • You have multiple items on a page
  • Consistent gutter/gap spacing is needed
  • You need a non-standard layout where columns can span multiple rows or rows can span multiple columns (e.g. a Bento Box)

Positioning

Standard Elements

The following utility classes can be used to position elements (ideally not grid or flex elements - these are better positioned using justification and alignment below). An element's position will be static by default.

Class Description
fixed Sets position to fixed (relative to the browser window)
absolute Sets position to absolute (outside the normal flow of the document)
relative Sets position to relative (to itself, in the normal flow of the document)
sticky Sets position to sticky (relative to a specified threshold, and fixed once that is offscreen)

Z-Index

To control the three-dimensional stacking order of elements, the following z-index utility classes can be used:

Class CSS
z-0 z-index: 0;
z-10 z-index: 10;
z-20 z-index: 20;
z-30 z-index: 30;
z-40 z-index: 40;
z-50 z-index: 50;
z-auto z-index: auto;

Custom values can also be set using z-[value], e.g. z-[100] to set the z-index to 100. We discourage using very large values unless absolutely necessary.
