Primary and secondary colours have five shades each: the main shade, and two lighter and darker shades. Monotone colours have five lighter and five darker shades.

Usage

The lighter and darker shades of the primary and secondary colours should only be used to complement the main shade or emphasise a transition state (e.g. a hover or active state on a button). They should not be used in isolation.

Syntax

To set the colour of an element, construct a class name using the following structure:

property-colourName[-shadeName]

The shade name is optional (without it, the main colour will be used), and the property may be one of the following:

  • text-
  • bg-
  • border-
  • outline-
  • decoration-
  • shadow-
  • fill- (SVG)
  • stroke- (SVG)

In practice, text, bg, and border will be the most commonly used, and some properties such as border allow for further specificity, e.g. border-l can be used to target an element's left border only.

Primary Colours

It is particularly important to stick with the default colour (rather than using lighter or darker shades) when applying the primary colours, so as not to dilute brand identity.

Shades should only be used if accessibility is compromised (as can be the case with Modern Green) or when using state changes.
