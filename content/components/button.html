Description

Buttons represent an action to be performed (such as visiting a link, downloading a document or submitting/resetting form data). The button's text and optional icon should clearly convey its semantic intent.

Syntax

Use the include tag to inject a button. Icons can be included by using properties; it is not necessary to embed them.

{{ html("
  {% include '@components/button/button.twig' with {
    property: 'value'
    ...
  } %}
  ")}}

Properties

A button has one mandatory property and fifteen optional properties, five of which relate to icons.

Variants

The button component is designed to be flexible and customisable, and thus includes a relatively large number of variants.

Text

Button text (btn_text) should be concise and clearly communicate the action the button will perform. If there is too little text, the action may be ambiguous, and if there is too much text, either use a link instead of a button, or shorten the text and include supporting information before or after the button.

{% include '@components/button/button.twig' with { btn_text: 'Download Prospectus', btn_type: 'button' } %}

Example code:

{{ html("
  {% include '@components/button/button.twig' with {
    btn_text: 'Download Prospectus'
  } %}
  ")}}

{{ html("

Type

The btn_type can either be link (the default), button, submit or reset. Its type will not alter its appearance but will alter the underlying markup used.

Recommended use cases for each are outlined below:

btn_type Use Case(s)
link CTA links, document downloads
button Any other functionality not covered by the other types, including code execution
submit Search buttons, submiting the contents of a form
reset Reset the contents of a form

Example code for a button to download a document:

{{ html("
  {% include '@components/button/button.twig' with {
    btn_text: 'Download Prospectus',
    btn_type: 'link',
    btn_url: '/path/to/download',
    is_download: true
  } %}
  ")}}

Icons

Icons should serve as a visual shortcut or hint as to the purpose of the button.

Icon Placement

Icons should generally be placed to the left of the button text. Although we allow for right-placement, icons placed to the right tend to be less visually useful, as they're scanned after the text (in left-to-right languages at least).

Example button with icon:

{% include '@components/button/button.twig' with { btn_text: 'Download Prospectus', btn_type: 'button', icon_name: 'download', icon_type: 'solid', icon_pos: 'left' } %}

Example code:

{{ html("
  {% include '@components/button/button.twig' with {
    btn_text: 'Download Prospectus',
    btn_type: 'button',
    icon_name: 'download',
    icon_type: 'solid',
    icon_pos: 'left'
  } %}
  ")}}

Situations in which right-placement may be more acceptable would be a button linking to an external site:

{% include '@components/button/button.twig' with { btn_text: 'Link to External Site', btn_type: 'button', icon_name: 'up-right-from-square', icon_type: 'solid', icon_pos: 'right' } %}

Icon-Only Buttons

In some cases you might need an icon-only button, in which case the icon is wrapped in a container with a title and aria label added for accessibility (the content of which is generated from the mandatory btn_text property). For these kinds of buttons it can also be useful to set the btn_padding to equal, to remove the extra horizontal btn_padding.

{% include '@components/button/button.twig' with { btn_padding: 'equal', btn_text: 'User Profile', btn_type: 'button', icon_name: 'user', icon_type: 'solid', is_icon_only: true, } %}

Example code:

{{ html("
  {% include '@components/button/button.twig' with {
    btn_padding: 'equal',
    btn_text: 'Icon Only',
    btn_type: 'button',
    icon_name: 'user',
    icon_type: 'solid',
    is_icon_only: true,
  } %}
  ")}}

Size and Width

Buttons come in three sizes: small, medium (the default) and large. The size can be set using the btn_size property.

{% include '@components/button/button.twig' with { btn_text: 'Button (S)', btn_type: 'button', btn_size: 'small' } %} {% include '@components/button/button.twig' with { btn_text: 'Button (M)', btn_type: 'button', btn_size: 'medium' } %} {% include '@components/button/button.twig' with { btn_text: 'Button (L)', btn_type: 'button', btn_size: 'large' } %}

Example code:

{{ html("
  {% include '@components/button/button.twig' with {
    btn_text: 'Button (S)',
    btn_type: 'button',
    btn_size: 'small'
  } %}
  ")}}

By default, a button will fit its content (including padding), but a button can also stretch to fill its container by setting btn_width to full (the value being fit by default).

    btn_text: 'Large Full-Width Button',
    btn_type: 'button',
    btn_size: 'large',
    btn_width: 'full'
  } %}
  ")}}

Other Properties

The final two properties not covered above are is_disabled and btn_modifiers; they are covered last as they should be the least likely to be used.

Disabling a Button

The is_disabled property is set to false by default. Setting it to true will add the disabled attribute to the button and modify its styling and cursor behaviour, and if the button is a link, the btn_url will be overridden.

{% include '@components/button/button.twig' with { btn_text: 'Disabled Button', btn_type: 'button', is_disabled: true } %}

Example code:

{{ html("
  {% include '@components/button/button.twig' with {
    btn_text: 'Disabled Button',
    btn_type: 'button',
    is_disabled: true
  } %}
  ")}}

Custom Styling

In what should only be exceptional circumstances, the nine provided button themes may not satisfy requirements (a different colour may be requested for example). In this case the btn_theme should be set to none, and new classes can be injected using the btn_modifiers property.

{% include '@components/button/button.twig' with { btn_modifiers: 'border-salmon text-salmon fill-salmon', btn_text: 'Custom Button', btn_type: 'button', btn_theme: 'none', icon_name: 'atom-simple', icon_type: 'regular', icon_size: 'w-5' } %}

Example code:

{{ html("
  {% include '@components/button/button.twig' with {
    btn_text: 'Disabled Button',
    btn_type: 'button',
    is_disabled: true
  } %}
  ")}}
