Description

Buttons represent an action to be performed (such as visiting a link, downloading a document or submitting/resetting form data). The button's text should clearly convey its semantic intent.

{% embed '@components/button/button.twig' with { text: 'Jump To Properties', url: '#properties', } %} {% block html_left %} {% include '@components/icon/icon.twig' with { name: 'circle-down', fill: 'fill-white', modifiers: 'mr-2' } %} {% endblock %} {% endembed %}

Syntax

Use the include tag to create a button, or the embed tag if you want to inject extra content (such as an icon). Use with the only keyword if you need to protect it from variables in the parent scope.

{{ html("
{% include 'ul:button' with {
  property: 'value'
  ...
} only %}
")}}

A button has one mandatory property and sixteen optional properties.

Variants

The button component is designed to be flexible and customisable, and thus includes a relatively large number of variants.

Text

Button text (text) should be concise and clearly communicate the action the button will perform. If there is too little text, the action may be ambiguous, and if there is too much text, either use a link instead of a button, or shorten the text and include supporting information before or after the button.

xToo short:

xToo long:
{% include '@components/button/button.twig' with { text: 'Download our Prospectus and See Where Your Future Takes You', type: 'button' } %}
âœ“Better:
{% include '@components/button/button.twig' with { text: 'Download Prospectus', type: 'button' } %}

Example code:

{{ html("
{% include 'ul:button' with {
  text: 'Download Prospectus'
} %}
")}}

The property text-classes can also be used to specifically target the text of the button. This could be useful in the case of the text being a different size or hidden at certain breakpoints (e.g. if using with an icon).

Type

The type can either be link (the default), button, submit or reset. Its type will not alter its appearance but will alter the underlying markup used.

Recommended use cases for each are outlined below:

type Use Case(s)
link CTA links, document downloads
button Any other functionality not covered by the other types, including code execution
submit Search buttons, submiting the contents of a form
reset Reset the contents of a form

Example code for a button to download a document:

{{ html("
{% include 'ul:button' with {
  text: 'Download Prospectus',
  type: 'link',
  url: '/path/to/download',
  is_download: true
} %}
")}}

Theme

A button's theme will help communicate the button's function (i.e. whether it's primary or secondary in the given context). There are five button themes: two solid, two outline and one transparent theme.

Applying a theme is straightforward. Below is an example of how to change the theme to solid-blue:

{{ html("
{% include 'ul:button' with {
  text: 'Solid Blue',
  theme: 'solid-blue'
} %}
")}}

Custom Styling

To add additional classes to the existing theme, use the modifiers property.

In certain circumstances, you may need a completely custom style. In these cases the overrides property can be used. Here's an example:

{% include '@components/button/button.twig' with { overrides: 'w-full px-4 py-3 border-2 border-salmon text-salmon fill-salmon hover:bg-salmon hover:text-white hover:fill-white active:bg-salmon-d1 transition cursor-pointer', text_classes: 'text-xl', text: 'Custom Button', type: 'button' } %}

Example code:

{{ html("
{% include 'ul:button' with {
  overrides: 'w-full px-4 py-2 border-2 border-salmon text-salmon fill-salmon hover:bg-salmon hover:text-white hover:fill-white transition cursor-pointer',
  text_classes: 'text-xl',
  text: 'Custom Button',
  type: 'button'
} %}
")}}

Icon Embeds

Content can be embedded into a button using the html_left or html_right properties. Icons serve as a good example, and should be used as a visual shortcut or hint as to the purpose of the button. It is recommended to set the icon's aria_hidden property to true when using icons within buttons.

Icon Placement

Icons should generally be placed to the left of the button text. Although we allow for right-placement, icons placed to the right tend to be less visually useful, as they're scanned after the text (in left-to-right languages at least).

Icon-only Buttons

In some cases you might need an icon-only button, in which case the text_classes property should be used to hide the text (it will still be used for the button title to ensure the icon remains accessible, unless the title is set, in which case that will be used).

For these kinds of buttons it can also be useful to customise the padding property, to ensure the horizontal and vertical values are equal.

{% embed '@components/button/button.twig' with { text: 'User', type: 'button', padding: 'p-4', text_classes: 'hidden' } %} {% block html_left %} {% include '@components/icon/icon.twig' with { name: 'user', type: 'solid', size: 'w-6', fill: 'fill-white', aria_hidden: true } only %} {% endblock %} {% endembed %}

Other Properties

The is_disabled property is set to false by default. Setting it to true will add the disabled attribute to the button and modify its styling and cursor behaviour, and if the button is a link, the url will be overridden.

{% include '@components/button/button.twig' with { text: 'Disabled Button', type: 'button', is_disabled: true } %}

Example code:

{{ html("
{% include 'ul:button' with {
  text: 'Disabled Button',
  type: 'button',
  is_disabled: true
} %}
")}}
